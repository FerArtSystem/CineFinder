CineFinder

Entregavel HTML/CSS com modelo Java reformulado

Versao atualizada - foco social + cinemas no mapa

1. Entidades Reformuladas (API Java)

Agora o banco Java persiste apenas o dominio social. O catalogo de conteudo ficou separado na API C#.

  • Usuario: cadastro e autenticacao (nome, email, nickname, senhaHash, dataCadastro).
  • Avaliacao: feedback social com nota, comentario, usuarioId e referencia externa (filmeId ou serieId).
  • Favorito: vinculo de usuario com conteudo externo via filmeId ou serieId.
  • Post: timeline da comunidade com conteudo, dataCriacao e dataEdicao.

Filme, Serie e Genero nao sao mais entidades JPA na API Java.

2. Repositorios Atuais

Entidade Repositorio Consultas especificas
Usuario UsuarioRepository findByEmail, existsByEmail, existsByNickname
Avaliacao AvaliacaoRepository findByFilmeId, findBySerieId, findByUsuarioId
Favorito FavoritoRepository findByUsuarioId
Post PostRepository findByUsuarioId, findAllByOrderByDataCriacaoDesc(Pageable)

3. Controllers e Endpoints (modelo vigente)

3.1 Controllers sociais

Controller Rota base Operacoes
UsuarioController /api/usuarios CRUD completo + POST /login
AvaliacaoController /api/avaliacoes CRUD completo + filtros por filme, serie e usuario
FavoritoController /api/favoritos GET listar, GET por id, POST criar, DELETE remover, GET por usuario

3.2 Controller agregador de cinemas

Controller Rota base Endpoints
CinemaController /api/cinemas GET /proximos?lat=&lng=&raio= e GET /cidade?q=&raio= (OpenStreetMap via Nominatim + Overpass)

4. import.sql no modelo novo

O import.sql da API Java carrega somente dados sociais para testes imediatos:

  • Usuarios de teste com senha hash BCrypt.
  • Avaliacoes com referencia externa de conteudo (filmeId e serieId).
  • Favoritos com referencia externa de conteudo.
  • Posts iniciais da comunidade.

Conclusao: a arquitetura atual separa claramente conteudo (API C#) e social/geolocalizacao (API Java), mantendo o frontend integrado as duas APIs.

5. APIs Externas Utilizadas

O projeto utiliza APIs externas para enriquecer catalogo e geolocalizacao, sem acoplar esses dados ao banco Java.

API/Servico Finalidade no projeto Onde e utilizada
TMDB API Busca de posters, backdrop, informacoes extras e catalogo complementar de filmes/series. Frontend (camada JS de integracao com chave TMDB).
Nominatim (OpenStreetMap) Geocodificacao de cidade para latitude/longitude na busca de cinemas por local. API Java em /api/cinemas/cidade e fallback no frontend.
Overpass API (OpenStreetMap) Consulta de pontos com amenity=cinema por coordenada e raio. API Java em /api/cinemas/proximos//api/cinemas/cidade e fallback no frontend.
Leaflet + OpenStreetMap Tiles Renderizacao do mapa interativo e marcadores de cinemas na tela. Frontend na rota #cinemas.
